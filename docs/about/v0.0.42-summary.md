## 🎯 概述

本次更新对 Faster APP 的配置系统进行了全面升级，引入了 **9 大核心功能**，使配置更安全、更灵活、更易用。

---

## 📦 核心功能（9 大功能）

### 1️⃣ 敏感信息保护 🔐

**问题**：密码、密钥等敏感信息容易在日志中泄露  
**解决方案**：使用 Pydantic `SecretStr` 类型

```python
from pydantic import SecretStr

class JWTSettings(BaseModel):
    secret_key: SecretStr = SecretStr("your-secret")
    
# JSON 序列化自动脱敏
print(settings.jwt.secret_key)  # SecretStr('***HIDDEN***')

# 需要时获取原始值
actual_key = settings.jwt.secret_key.get_secret_value()
```

---

### 2️⃣ TORTOISE_ORM 动态配置 🔄

**问题**：在 `__init__` 中生成配置，无法反映运行时更新  
**解决方案**：改为 `@property` 动态生成

```python
class Settings(BaseSettings):
    @property
    def TORTOISE_ORM(self) -> dict:
        """动态生成配置，始终使用最新值"""
        return {
            "connections": {
                "POSTGRES": {
                    "credentials": {
                        "host": self.database.host,  # 动态读取
                        "password": self.database.password.get_secret_value(),
                    }
                }
            }
        }
```

---

### 3️⃣ 生产环境配置验证 ✅

**问题**：生产环境配置错误导致安全隐患  
**解决方案**：字段级 + 模型级验证器

```python
@field_validator("port")
def validate_port(cls, v):
    if not 1 <= v <= 65535:
        raise ValueError(f"端口必须在 1-65535 之间")
    return v

@model_validator(mode="after")
def validate_production_settings(self):
    if not self.debug:  # 生产环境
        if len(self.jwt.secret_key.get_secret_value()) < 32:
            raise ValueError("生产环境密钥长度必须 >= 32")
    return self
```

---

### 4️⃣ 环境变量前缀 🏷️

**问题**：多应用环境变量冲突  
**解决方案**：统一使用 `FASTER_` 前缀

```bash
# 避免冲突
export FASTER_DATABASE__HOST=localhost  # ✅
export FASTER_SERVER__PORT=8000         # ✅

# 而不是
export DATABASE_HOST=localhost  # ❌ 可能冲突
```

---

### 5️⃣ 配置分组与嵌套 📦

**问题**：所有配置扁平化，难以管理  
**解决方案**：按功能模块分组

```python
class Settings(BaseSettings):
    server: ServerSettings      # 服务器配置
    jwt: JWTSettings            # JWT 配置
    database: DatabaseSettings  # 数据库配置
    log: LogSettings            # 日志配置

# 访问方式
settings.server.host
settings.database.port
settings.jwt.secret_key
```

---

### 6️⃣ 配置文件重构 🗂️

**问题**：所有配置在一个文件中，难以维护  
**解决方案**：拆分到独立文件

```
faster_app/settings/groups/
├── __init__.py      # 统一导出
├── server.py        # ServerSettings
├── jwt.py           # JWTSettings  
├── database.py      # DatabaseSettings
└── log.py           # LogSettings
```

---

### 7️⃣ DATABASE_URL 支持 🔗

**问题**：数据库配置需要多个环境变量  
**解决方案**：单个 URL 字符串

```bash
# 新方式（推荐）- 1 个变量
export FASTER_DATABASE_URL=postgresql://user:pass@host:5432/db

# 旧方式（仍支持）- 6 个变量
export FASTER_DATABASE__TYPE=postgres
export FASTER_DATABASE__HOST=host
export FASTER_DATABASE__PORT=5432
export FASTER_DATABASE__USER=user
export FASTER_DATABASE__PASSWORD=pass
export FASTER_DATABASE__DATABASE=db
```

**优势**：
- ✅ 符合 12-Factor App 最佳实践
- ✅ 云平台友好（Heroku、Railway、Render）
- ✅ 简化部署配置
- ✅ 完全向后兼容

---

### 8️⃣ 数据库 Schema 支持 🏢

**问题**：多租户应用数据隔离困难  
**解决方案**：支持 PostgreSQL Schema

```bash
# 租户 A
export FASTER_DATABASE_URL=postgresql://app:pass@db:5432/saas?schema=tenant_a

# 租户 B
export FASTER_DATABASE_URL=postgresql://app:pass@db:5432/saas?schema=tenant_b
```

**典型场景**：
- 多租户 SaaS 应用（每租户一个 schema）
- 微服务数据隔离（每服务一个 schema）
- 环境数据隔离（dev/test/staging）

---

### 9️⃣ 配置字段命名规范 📝

**问题**：字段命名不统一  
**解决方案**：统一使用 `snake_case`

```python
# ✅ 推荐（新）
class ServerSettings(BaseModel):
    host: str = "0.0.0.0"
    port: int = 8000

# ❌ 避免（旧）
class ServerSettings(BaseModel):
    HOST: str = "0.0.0.0"
    PORT: int = 8000
```

## 🚀 快速开始

### 1. 使用 DATABASE_URL

```bash
# 开发环境
export FASTER_DATABASE_URL=sqlite:///dev.db

# 生产环境
export FASTER_DATABASE_URL=postgresql://app:secret@prod.db:5432/myapp
```

### 2. 多租户 SaaS

```bash
# 租户 A
docker run -e FASTER_DATABASE_URL="postgresql://app:pass@db:5432/saas?schema=tenant_a" myapp

# 租户 B
docker run -e FASTER_DATABASE_URL="postgresql://app:pass@db:5432/saas?schema=tenant_b" myapp
```

from faster_app.settings import configs
