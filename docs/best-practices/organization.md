# 项目组织最佳实践

本页面介绍如何合理组织 Faster APP 项目结构。

## 模块划分原则

### 按业务功能划分

```
apps/
├── auth/          # 认证授权
├── users/         # 用户管理
├── posts/         # 文章管理
├── comments/      # 评论管理
└── notifications/ # 通知系统
```

### 按技术层次划分（不推荐）

```
# ❌ 不推荐这种结构
apps/
├── models/
├── views/
└── services/
```

## 模块大小控制

- **小型项目**（< 10 个模型）：3-5 个模块
- **中型项目**（10-50 个模型）：8-15 个模块
- **大型项目**（> 50 个模型）：按业务域划分
